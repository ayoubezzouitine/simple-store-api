### 🛒 Store API – Spring Boot Backend

This project is a **RESTful backend application** built with **Spring Boot**, designed to manage an online store system. It provides a structured and scalable architecture for handling core e-commerce functionalities such as users, products, categories, messaging, and notifications.

---

### 🚀 Features

* 👤 **User Management**

  * Create and manage user profiles
  * Handle authentication-related data (extendable)

* 📦 **Product Management**

  * CRUD operations for products
  * Product categorization

* 🗂️ **Category Management**

  * Organize products into categories

* 📍 **Address Handling**

  * Manage user addresses for delivery and billing

* 💬 **Messaging System**

  * Internal messaging between users or system notifications

* 🔔 **Notification System**

  * Supports notifications (e.g., email notifications)

---

### 🏗️ Architecture

The project follows a clean layered structure:

* `controllers` → Handle HTTP requests
* `services / controlleradvice` → Business logic & global exception handling
* `entities` → JPA entities (User, Product, Category, Address, Message, etc.)
* `dtos` → Data Transfer Objects
* `mappers` → Entity-DTO conversions
* `exceptions` → Custom exception handling

---

### 🛠️ Tech Stack

* Java
* Spring Boot
* Spring Data JPA
* Hibernate
* REST APIs
* Maven / Gradle (depending on your setup)

---

### 🎯 Goal

The goal of this project is to build a **modular, maintainable, and scalable backend** for an e-commerce platform, following best practices in Spring Boot development.
